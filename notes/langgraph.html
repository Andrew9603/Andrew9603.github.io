LangGraph ​

是什么？ ​

LangGraph 是一个用于构建有状态的、多参与者 LLM 应用的框架。它在 LangChain 的基础上引入了基于图的编程模型——节点代表计算步骤，边定义执行流程。这使它非常适合构建复杂的 Agent 工作流。

解决什么问题？ ​

用简单的 chain-of-thought 提示来构建生产级 Agent 系统有几个局限性：

  • 状态管理 — 跨步骤跟踪对话状态、中间结果和上下文容易出错
  • 循环工作流 — Agent 通常需要循环、重试或根据条件分支；线性链无法表达这些
  • 人在回路 — 生产系统需要检查点来支持人工审核和批准
  • 可观测性 — 调试多步 Agent 工作流需要对每一步有清晰的可见性

LangGraph 通过其基于图的架构解决了所有这些问题。

核心流程 ​

一个 LangGraph 应用被定义为一个状态图：

  1. 定义状态 — 创建一个类型化的状态 schema，在图中流转
  2. 创建节点 — 每个节点是一个读取和更新状态的函数
  3. 定义边 — 用条件边或无条件边连接节点
  4. 编译 — 将图构建为可执行的应用
  5. 运行 — 用初始状态调用图
python
from langgraph.graph import StateGraph

graph = StateGraph(AgentState)
graph.add_node("agent", agent_node)
graph.add_node("tools", tool_node)
graph.add_edge("agent", "should_continue")
graph.add_conditional_edges("should_continue", router)
app = graph.compile()

核心概念 ​

状态（State）： 在图中流转的核心数据结构。每个节点接收当前状态并返回更新。状态通常是 TypedDict 或 Pydantic 模型。

节点（Nodes）： 执行计算的函数。每个节点接收状态、处理它，并返回部分状态更新。节点可以调用 LLM、调用工具或执行任意逻辑。

边（Edges）： 节点之间的连接。可以是无条件的（总是从 A 到 B）或有条件的（根据状态值路由）。

循环（Cycles）： 与 DAG 不同，LangGraph 支持循环。这对于 Agent 循环至关重要——Agent 需要反复推理、行动和观察，直到任务完成。

检查点（Checkpoints）： 内置的持久化机制，在每一步保存图的状态。支持时间旅行调试、人在回路工作流和容错。

子图（Subgraphs）： 图可以嵌套在其他图中，支持模块化设计和通用工作流的复用。
