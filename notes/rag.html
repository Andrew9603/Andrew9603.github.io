RAG（检索增强生成） ​

是什么？ ​

RAG（Retrieval-Augmented Generation）是一种通过在生成回答之前从外部知识库检索相关信息来增强 LLM 回答质量的技术。它不是完全依赖模型的参数记忆，而是在查询时提供最新的、领域特定的上下文。

解决什么问题？ ​

LLM 有几个固有的局限性，RAG 可以解决这些问题：

  • 知识截止 — 模型的训练数据有截止日期；RAG 可以提供最新信息
  • 幻觉 — 模型可能生成看似合理但错误的信息；RAG 让回答基于真实文档
  • 领域专业性 — 通用模型缺乏专业领域的深度知识；RAG 可以注入领域特定的上下文
  • 来源追溯 — RAG 可以将回答追溯到具体文档，支持引用和验证

核心流程 ​

一个标准的 RAG 流水线包含两个主要阶段：

索引阶段 ​

  1. 文档加载 — 从各种来源（PDF、网页、数据库）加载文档
  2. 分块 — 将文档切分为更小的、语义连贯的片段
  3. 向量化 — 使用 Embedding 模型将片段转换为向量表示
  4. 存储 — 将向量连同元数据一起存入向量数据库

查询阶段 ​

  1. 查询向量化 — 将用户问题转换为向量
  2. 检索 — 从向量数据库中找到最相似的片段
  3. 上下文组装 — 将检索到的片段组合成 prompt
  4. 生成 — LLM 基于组装好的上下文生成回答
文档 → 分块 → 向量化 → 向量数据库
                              ↓
用户查询 → 向量化 → 检索 → 上下文 → LLM → 回答

核心概念 ​

分块策略（Chunking）： 文档的切分方式显著影响检索质量。可选方案包括固定大小分块、基于句子的切分、递归字符切分和语义分块。正确的策略取决于数据和使用场景。

Embedding 模型： 如 text-embedding-ada-002、bge-large、e5 等模型将文本转换为稠密向量。模型的选择直接影响检索精度。

混合检索（Hybrid Retrieval）： 将稠密检索（向量相似度）与稀疏检索（BM25 关键词匹配）结合使用，通常优于单独使用任一种方法。

重排序（Reranking）： 在初始检索之后，使用重排序模型（如 Cohere Rerank、BGE Reranker）重新排列结果以提高相关性。

元数据过滤（Metadata Filtering）： 为片段添加元数据（来源、日期、类别），在相似度搜索之前先缩小结果范围。

后续探索方向 ​

  • 高级分块策略（语义分块、命题级分块）
  • 查询变换技术（HyDE、step-back prompting）
  • 基于知识图谱的 Graph RAG
  • RAG 系统评估指标（忠实度、相关性、召回率）
  • 生产环境考量：缓存、限流与成本控制
