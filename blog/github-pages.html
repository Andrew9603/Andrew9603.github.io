使用 GitHub Pages 搭建博客 ​

本文记录使用 VitePress + GitHub Pages 搭建个人技术博客的完整流程。

为什么选择 VitePress？ ​

VitePress 是基于 Vite 构建的静态站点生成器，速度快、配置简单，专门为文档和博客类内容设计。主要优势包括：

  • Markdown 优先，内置 Vue 组件支持
  • 快速的开发服务器，支持热模块替换
  • 优化的静态站点生成，配置量极少
  • 内置搜索、导航和主题功能

项目初始化 ​

创建新项目目录并初始化：

bash
mkdir my-blog && cd my-blog
npm init -y
npm install -D vitepress

创建 docs 目录结构：

docs/
├── .vitepress/
│   └── config.js
├── index.md
├── about.md
└── blog/
    └── first-post.md

在 package.json 中添加脚本：

json
{
  "scripts": {
    "docs:dev": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:preview": "vitepress preview docs"
  }
}

运行 npm run docs:dev 启动开发服务器。

VitePress 配置 ​

docs/.vitepress/config.js 文件控制站点的导航、侧边栏和元数据：

js
import { defineConfig } from 'vitepress'

export default defineConfig({
  title: "我的技术博客",
  description: "技术笔记与工程实践",
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      { text: '博客', link: '/blog/first-post' },
      { text: '关于', link: '/about' }
    ],
    socialLinks: [
      { icon: 'github', link: 'https://github.com/你的用户名' }
    ]
  }
})

GitHub Pages 部署 ​

仓库设置 ​

在 GitHub 上创建一个名为 你的用户名.github.io 的仓库。这个命名约定会自动启用 GitHub Pages 托管。

GitHub Actions 工作流 ​

创建 .github/workflows/deploy.yml：

yaml
name: 部署 VitePress 到 GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
      - run: npm ci
      - run: npm run docs:build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment

GitHub 设置 ​

  1. 进入仓库 Settings → Pages
  2. 将 Source 设置为 GitHub Actions
  3. 推送到 main 分支即可触发部署

编写内容 ​

所有内容使用 Markdown 编写。VitePress 支持标准 Markdown 以及一些扩展，比如代码组、自定义容器和 Vue 组件。

博客文章的 frontmatter 示例：

markdown
---
title: 我的第一篇文章
date: 2024-01-01
---

# 我的第一篇文章

内容写在这里...

总结 ​

整个搭建过程只需要几步：初始化项目、配置 VitePress、编写 Markdown 内容、设置 GitHub Actions 工作流，然后推送到仓库。每次推送到 main 分支，博客都会自动部署。
